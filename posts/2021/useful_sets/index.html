Useful things you can do with Sets in Python¬∂

Perhaps Python‚Äôs most underrated built-in

I don‚Äôt use Sets as often as I should¬∂

I have noticed that everytime I need to create a collection/data structure in Python, I almost always default to:

  • a list (about 90% of the time)

  • a numpy array (when I‚Äôm thinking about speed)

  • a dictionary (when I want a hashmap)

And I don‚Äôt think about using Sets as much as I really should, which is a real pity because Sets are a very special and powerful Python built-in type that can solve many common problems. So here‚Äôs short little blog post on some useful things you can do with Sets.

Removing duplicate elements¬∂

A set can only contain unique elements, and this property makes sets a very convenient data structure for removing duplicates.

names = ["Andy", "Bob", "Andy", "Jay", "Peter", "Bob", "Russ", "Jay"]
unique_names = set(names)
print(unique_names)

{'Peter', 'Jay', 'Russ', 'Andy', 'Bob'}

It is common to apply the set() function on a Python iterable, such as a List (ie. example above) or a Tuple, but don‚Äôt forget that String is also an iterable type in Python! This means we can also use sets to find the unique letters in any given string.

long_string = "supercalifragilisticexpialidocious"
unique_letters = set(long_string)
print(unique_letters)

{'o', 'g', 's', 'd', 't', 'l', 'e', 'a', 'f', 'x', 'u', 'i', 'r', 'c', 'p'}

Set Theory¬∂

The concept of a Set in Python comes from Set Theory, which is a foundational branch of Mathematics that teaches us how we can think about collections of objects or numbers. Probably, Set Theory was your 1st introduction to the famous Venn Diagram, and it‚Äôs related concepts such as Unions and Intersections. Just like what we learnt in Math class, we can perform the following useful operations on 2 or more Python Sets.

Union¬∂

Join 2 or more Sets and return the non-duplicated results

garden1 = set(['tulips', 'hydrangeas', 'roses', 'chrysanthemums'])
garden2 = set(['chrysanthemums','lilies','orchids','hydrangeas'])
garden3 = set(['roses','lilies', 'daisies'])

unique_flowers = garden1.union(garden2,garden3)
print(unique_flowers)

{'tulips', 'daisies', 'hydrangeas', 'orchids', 'lilies', 'roses', 'chrysanthemums'}

Intersection¬∂

Return only the common elements in 2 or more Sets

# Each group contains a list of customer_ids
group_a = set([121, 34, 156, 22, 95, 61])
group_b = set([34, 156, 31, 24, 53, 99, 22])
group_c = set([24, 99, 84, 22, 14, 34, 53])

common_customer_ids = group_a.intersection(group_b, group_c)
print(common_customer_ids)

{34, 22}

Simple is better than Complex

Looking back at my old code, there are so many occasions where I used more complex methods, such as merging 2 DataFrames with an inner join, to find an intersecting elements.

In many cases, we are better served by using the much simpler approach of calling the intersection method between 2 sets instead.

Difference¬∂

Return only the non-common elements between 2 or more sets

languages1 = set(['Python','C++','Golang','Haskell', 'R'])
languages2 = set(['R', 'Javascript', 'Python', 'Julia', 'Erlang'])

non_common_languages = languages1.difference(languages2)
print(non_common_languages)

{'Haskell', 'Golang', 'C++'}

Conditional Checks¬∂

I also categorize several Set methods under the purpose of performing some type of conditional check and returning either a True or False boolean. These are particularly useful and fast operations to add to long scripts or unit tests.

Isdisjoint¬∂

Returns True if no elements between 2 Sets are common, and False if there is at least 1 common element.

colours1 = set(['red', 'yellow', 'green'])
colours2 = set(['blue', 'purple', 'lilac'])

colours1.isdisjoint(colours2)

True

colours3 = set(['red', 'yellow', 'green', 'rainbow'])
colours4 = set(['blue', 'purple', 'lilac', 'rainbow'])

colours3.isdisjoint(colours4)

False

Issubset and Issuperset¬∂

In this signature primary.method(secondary):

Issubset returns True if all the elements of the primary set are members of the secondary set, returns False otherwise.

Issuperset returns True if all the elements of the secondary set are members of the primary set, return False otherwise.

primary = set(['a','b'])
secondary = set(['a','b','c','d'])

primary.issubset(secondary)

True

primary = set(['x','y','z'])
secondary = set(['x','y'])

primary.issuperset(secondary)

True

Special use cases¬∂

Lastly, there are a lot of special use cases that are neat and novels ways to use Sets.

Speedy check for membership¬∂

Under the hood, sets are implemented using hash tables. This means when we want to check if an element is a member of a set (ie. lookup), this operation is faster versus a similar membership check in a list.

Source: https://stackoverflow.com/questions/8929284/what-makes-sets-faster-than-lists/8929445

big_list = range(10**8)
big_set = set(range(10**8))

%%timeit
1612 in big_list

The slowest run took 15.21 times longer than the fastest. This could mean that an intermediate result is being cached.
10000000 loops, best of 5: 71.2 ns per loop

%%timeit
1612 in big_set

10000000 loops, best of 5: 35.5 ns per loop

Budget load balancer¬∂

1 of Set‚Äôs properties is that it‚Äôs elements are always unordered, and this is usually seen as a disadvantage. In fact, it‚Äôs surprising that Set actually has a pop() method that is usually reserved for ordered data structures. When you call pop() on a set, it removes a random element. I used to think that was a pretty useless method, until I read this StackOverflow post.

Amusingly, this random removal of an element works surprisingly well to create a simple load balancer.

jobs = set(['job1', 'job2', 'job3','job4','job5'])

while jobs:
    current_job = jobs.pop()
    print(f'Processing {current_job}')

print('\nJobs done!')

Processing job2
Processing job1
Processing job3
Processing job4
Processing job5

Jobs done!
